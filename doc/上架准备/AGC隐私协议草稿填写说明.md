# AGC 隐私协议草稿填写说明（遨游轨迹）

适用于 AGC 后台「隐私协议 - 草稿」中「我们如何收集和使用您的个人信息」的填写。

---

## 一、1.1 选哪种写法

本应用**没有**在应用内提供「基本功能 / 全量功能」切换入口，建议**不勾选第一个 1.1**，改用下面第二种 1.1 的写法。

---

## 二、推荐填写方式

### 1. 第一个 1.1（不勾选）

- 若不勾选「我们为您提供了基本功能服务和全量功能服务」，则无需填写「应用内服务模式设置页面路径」和两个「请输入产品功能」。
- 直接使用下方第二种 1.1 即可。

### 2. 第二个 1.1（勾选并填写）

- **勾选**：「我们为您提供下述 [请输入产品功能] 在您使用相关业务的过程中，我们会处理所必需的信息，以便 [请选择]」

- **「请输入产品功能」** 建议填写（可微调表述）：

  **轨迹记录、打卡点记录、地图展示、华为帐号登录与云端同步、个人资料与意见反馈**

- **下拉「请选择」**：选与「提供核心功能 / 实现业务功能」最接近的选项（如「提供核心功能服务」「实现产品功能」等，以 AGC 当前可选为准）。

- 说明文案会变成：若您不提供相关信息，会影响到您使用本应用的相关功能/服务。可保留不改。

### 3. 1.2 基于履行法定义务（勾选并增加数据）

- **勾选**：「基于履行法定义务或其他法律法规规定的情形，我们可能会处理您的以下个人信息」

- 点击 **「增加数据信息」**，按实际收集情况添加，例如：

  | 数据类型（示例名称） | 使用目的/场景说明（示例）     |
  |----------------------|------------------------------|
  | 位置信息             | 用于轨迹记录、打卡点与起终点设置、地图展示 |
  | 华为帐号信息         | 用于登录、帐号标识与云端数据同步         |
  | 昵称、家族、头像等   | 用于个人资料展示与轨迹作者信息（若展示） |

- 若还有设备信息、日志信息等（如用于安全或合规），可再增加对应项并简要说明用途。

### 4. 「为了 实现应用功能」及 0/500 补充说明（重要）

- **产品功能**：若表单中 1.1 的「请输入产品功能」被截断，请补全为完整一句，例如：  
  **轨迹记录、打卡点记录、地图展示、华为帐号登录与云端同步、个人资料与意见反馈**

- **「在获取您的同意后」**：保持选择「在获取您的同意后」即可。

- **「我们需要」后的下拉（收集时机）**：  
  - 若本应用**仅在用户主动录制轨迹、设置起点/打卡点/终点时**在前台获取位置，**不要**选「在后台运行时」。  
  - 请改为选「在您使用相关功能时」「在前台使用应用时」等表示**前台、使用相关功能时**的选项，避免被认定为后台持续收集位置。

- **下方列出的数据类型**（GPS 位置、精确/大致位置、用户标识符、账号信息、图片或视频、文字信息、应用基本信息）：按实际勾选；若无需「大致位置」或「后台位置」，可在对应项或「增加数据信息」中删减/不勾选。

- **0/500 字的补充说明框**：建议填写一句澄清收集场景，例如（约 80 字）：  
  **位置信息仅在您主动进行轨迹记录、设置起点/打卡点/终点时于前台收集，用于绘制轨迹与展示地图，不在后台持续收集。账号与个人资料用于登录、云端同步及个人中心展示。**

## 四、2. 设备权限调用

此处需逐项说明应用为何申请、在何种场景下使用该权限（每项 500 字以内）。

### 位置（获取设备位置信息 / 模糊位置 / 后台位置）

- **建议填写**（按实际使用勾选并保留对应句，可删减）：
  - 本应用在前台录制轨迹时，会获取您的**精确位置**，用于记录轨迹路线、设置起点/打卡点/终点及在地图上展示。仅在您主动点击「开始记录」「设置起点」「设置打卡点」「设置终点」等操作时使用，不会在后台持续获取位置。
  - 若您仅申请了「模糊位置」：可将“精确位置”改为“位置信息（模糊位置）”，并说明用于大致展示轨迹范围或附近打卡点。
  - 若确实有**后台位置**（如持续记录轨迹）：需单独说明“在您开启轨迹录制且应用处于后台时，我们会持续获取位置以完成轨迹记录，录制结束或暂停后即停止”。

- **示例（约 180 字，可照抄或微调）**：
  > 用于轨迹记录与打卡点功能。在您使用「记录轨迹」「设置起点」「设置打卡点」「设置终点」时，我们仅在前台获取设备位置信息，用于绘制轨迹路线、保存打卡点位置并在地图上展示。不会在后台持续获取您的位置。

### 图片和视频（保存到用户公共目录）

- 若应用会**保存图片/视频到相册或公共目录**（如分享图保存、打卡点照片保存等），需在此说明用途与场景。
- **建议填写**（约 120 字）：
  > 用于打卡点图文记录与分享。在您为打卡点添加照片、或将轨迹分享图保存到本地时，应用会请求将图片保存到您的设备相册（公共目录），便于您查看与分享。仅在您主动选择「保存」时执行，不会自动保存。

- 若应用**从不保存图片/视频到公共目录**，可在此说明“本应用不将图片或视频保存至设备公共目录”，或在该权限项选择不添加/删除该条。

### 添加其他权限

- 若还申请了**存储、相机、网络**等权限，点击 **「添加权限及使用说明」** 增加对应项，每项用一两句话说明使用场景（例如：相机用于打卡点拍照；网络用于登录与云端同步）。

## 五、6. 管理您的个人信息

以下路径与「涉及个人信息种类」按遨游轨迹当前结构填写，请与应用内实际入口一致。

### 6.1 信息访问

- **应用内服务模式设置页面路径**：**我的 → 编辑个人信息**
- **涉及个人信息种类**：**昵称、所属家族、头像、个人简介；轨迹记录、打卡点及起终点信息可在「我的 → 我的轨迹」进入对应轨迹详情中查看**

（若希望简写，可只填：**我的 → 编辑个人信息** / **昵称、家族、头像、个人简介**；轨迹类单独一句或放在 6.2/6.3。）

### 6.2 信息更正

- **应用内服务模式设置页面路径**：**我的 → 编辑个人信息**
- **涉及个人信息种类**：**昵称、所属家族、头像、个人简介**；轨迹名称、打卡点名称等可在**我的 → 我的轨迹 → 进入对应轨迹详情**中编辑

### 6.3 信息删除

- **应用内服务模式设置页面路径**：**我的 → 我的轨迹 → 进入某条轨迹 → 轨迹详情页**（删除整条轨迹）；**我的 → 编辑个人信息**（不涉及删除账号时，仅修改或清空资料）
- **涉及个人信息种类**：**轨迹记录、打卡点及图文内容**（在轨迹详情中删除轨迹或打卡点）；**昵称、家族、头像等个人资料**（在编辑个人信息中修改或清空）

（若表单只允许一处路径，可填：**我的 → 我的轨迹 → 轨迹详情**，种类填：**轨迹及打卡点数据、个人资料**。）

### 6.4 撤销同意

- **涉及个人信息种类**：**位置信息、华为帐号信息、昵称/家族/头像等个人资料**
- **应用内服务模式设置页面路径**：**我的 → 编辑个人信息**（修改或停止使用个人资料）；**我的**（通过退出登录停止使用帐号与云端数据）。若应用内有「隐私设置」或「权限管理」入口，可改为该路径

### 6.5 注销服务

- **应用内服务模式设置页面路径**：**我的 → 注销账号**  
  若当前版本仅有「退出登录」、暂无「注销账号」入口，可填：**我的 → 退出登录**，并在应用内或隐私政策正文中说明账号注销方式（如联系客服或后续版本提供）。

遨游轨迹是一款专为记录与传承扫墓路线、墓地点信息而设计的鸿蒙应用。山上扫墓往往道路复杂、祖墓难寻，仅靠口口相传容易遗忘。本应用帮助您系统记录每次扫墓的行走路径与每个墓地的位置与故事，让后代也能按图索骥、延续祭祖文化。

## 附录 C：备案信息校验不通过

若 AGC 提示 **「备案信息校验不通过」**，常见原因与处理如下。

### 原因 1：社会信用代码填写不正确

- **适用**：主办单位类型为**企业**或**机构**时，需填**统一社会信用代码**（18 位）。
- **处理**：
  - 核对主办单位名称与工信部备案主体名称**完全一致**（无多余空格、无简写）。
  - 统一社会信用代码从营业执照或「全国企业信用信息公示系统」等官方渠道获取，逐字核对无输错。
  - 若主办单位为**个人**，此处应填**身份证号**（18 位），不是社会信用代码。

### 原因 2：包名未履行备案手续

- **含义**：应用包名（如本应用的 **com.lyf.aoyou**）需在工信部备案系统中完成**应用/互联网信息服务备案**，且鸿蒙版与安卓版包名需**分别**备案（若同时上架双平台）。
- **处理**：
  1. 登录 **工信部备案管理系统**（或您所在省份的备案入口），查看当前主体下是否已为 **com.lyf.aoyou**（鸿蒙应用）办理备案或是否需新增「应用备案」。
  2. 若尚未为该包名备案，需按当地通信管理局要求提交应用备案，备案通过后再在 AGC 重新提交或重新校验。
  3. 若只有鸿蒙版、无安卓版，只需确保 **com.lyf.aoyou** 对应的鸿蒙应用已完成备案即可。

具体备案流程、所需材料及入口以工信部/各省通信管理局最新规定为准；AGC 页面「了解更多」可跳转平台侧说明。
